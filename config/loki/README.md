grafana loki is a system to handle the logs in mini-BaaS. It is used to collect logs from all services and make them available for querying and visualization in Grafana.

## Why ?

Loki is a log aggregation system that is designed to be cost-effective and easy to operate. It is a good fit for mini-BaaS because it can handle the large volume of logs generated by the microservices architecture, while keeping operational overhead low.
with loki we can :

- collect errors API logs
- collect internal service logs
- correlate logs with metrics and traces in Grafana
- Debug issues in production with rich log data
- observe latency and error patterns across services
- obseerve usage patterns and user behavior through logs
- observe anomalies and potential security issues through logs
