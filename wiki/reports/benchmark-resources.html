Latency — what each costs per read

Both serve the identical PostgREST GET /rest/v1/bench_items?limit=30 read, n=60, warm, sequential. Grobase wins the p95 tail (2.2 ms vs 2.57 ms, 0.37 ms better). The p50 is a near-tie: Supabase edges it by just 0.12 ms (1.51 vs 1.63 ms) — inside same-box noise on n=60, not a real lead either way.

Resources — what each stack costs at rest

Same capability surface, very different holding cost. Grobase essential (the full-feature, Postgres-engine tier — the apples-to-apples comparable for Supabase self-host) idles at 821.7 MiB; Supabase self-host's 13 containers sum to 2884 MiB — 3.5× heavier. RSS = sum of docker stats resident memory.

Where Supabase's 2,884 MiB goes

Per-container RSS from docker stats (artifacts/bench/supabase-footprint-breakdown.txt). The API gateway (kong) alone holds 1563 MiB — 54% of the whole stack — before any of the 12 other services. This is the structural weight Grobase avoids by collapsing the gateway + planes into a lean Rust/Go core.

Grobase grows on one codebase — the footprint ladder

The same SDK and codebase scale from a 2.008 MiB single binary up the tier ladder, no rewrite. Even max (all engines + analytics) at 3,634 MiB is a deliberate, opt-in shape — you pay for footprint only when you turn on engines. Measured idle RSS per tier (artifacts/footprint-<tier>.json, 2026-06-12).

Honest verdict

What the numbers say, and where the comparison turns the other way.

Grobase wins — footprint 3.5× lighter
Same capability surface at 821.7 MiB vs 2884 MiB. kong alone (1563 MiB) is heavier than most of Grobase essential.
Grobase wins — read tail p95 2.2 ms
Grobase's read p95 (2.2 ms) beats Supabase (2.57 ms) by 0.37 ms on the identical n=60 probe.
Near-tie — read median Δ 0.12 ms
Supabase edges read p50 (1.51 vs 1.63 ms) — 0.12 ms, inside same-box noise on n=60. Not a real win for either side; shown truthfully.
Grobase wins — density 0 pools
24,888 tenants in ~2.9 MiB data-plane RSS with 0 idle pools (gate m46). Supabase self-host is one project per backend — density is not its model.
Supabase leads — hosted maturity their lead
As a managed product Supabase has a polished Studio, a deeper ecosystem, global edge functions, and third-party SOC2/HIPAA attestations. Grobase managed-cloud is mid-go-live.
Apples-to-apples same box
Both ran the identical PostgREST GET on one 20-vCPU box, n=60, 2026-06-13. The capability matrix confirms it is not a feature-stripped comparison.
