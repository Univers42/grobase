3 · The latency near-tie — shown truthfully

The most-asked metric, and the one place the answer isn't a clean win. At p50, Supabase is 0.12 ms faster (1.51 vs 1.63 ms) — a difference inside same-box run-to-run noise on n=60; we call it a TIE, not a Supabase win and certainly not a fabricated Grobase win. At the tail (p95) Grobase pulls ahead: 2.2 < 2.57 ms. Both run the identical PostgREST workload against a 500-row bench_items table.

4 · Footprint & density — the structural wins

Grobase's essential tier idles at 3.5x lighter than Supabase self-host (822 vs 2,884 MiB across 13 containers — kong alone is 1.526 GiB). And because isolation is per-request (not pool-per-tenant), data-plane RAM is decoupled from tenant count: flat at ~3 MiB to 24,887 tenants. Supabase self-host is one Postgres backend per project — its per-tenant holding cost grows linearly. (* single-binary floor compares nano to PocketBase; Supabase has no single-binary self-host at all.)

6 · Where Supabase still wins

The honesty rule, stated plainly. One is a measured metric (read p50, by 0.12 ms — a tie). The rest are hosted-product / certification leads, not benchmark metrics: managed-cloud maturity, Studio polish, ecosystem, global edge, and third-party attestation.

Read p50 latency by 0.12 ms
1.51 vs 1.63 ms on the same probe (n=60). Marginal and inside run-to-run noise — a genuine tie, but on the raw number Supabase is ahead. We will not invent a p50 win.
Managed-cloud maturity hosted product
Sign up → project → API key → Stripe-billed, all turnkey today. Grobase's B1–B6 components are built + gate-proven but flag-OFF; hosted go-live (Track B7) is in progress, not shipped.
Studio polish hosted console
Full hosted Studio (table editor, SQL editor, logs, GraphiQL). binocle-one's admin UI at /_/ is capable but not Studio-class. Honest gap on a managed metric — not a measurable benchmark.
Ecosystem & community adoption
Large community, many integrations, tutorials, StackOverflow gravity. Grobase is new. Same open core (gotrue/postgrest/kong) means many Supabase guides transfer — but reach is earned over time.
Global edge functions Deno edge
Edge Functions run on Deno Deploy's global edge network. We trade global edge for no invocation cap + data residency; multi-region function distribution is Track-C-deepen, not shipped.
Third-party SOC 2 / HIPAA attestation
Externally attested SOC 2 + ISO 27001 (Team) and a HIPAA BAA (Enterprise). Grobase ships re-verifiable controls (tamper-evident audit m141), but external attestation is an audit engagement ($$), not code.
