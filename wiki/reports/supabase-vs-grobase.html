4 · Where Grobase wins

Each with its artifact / gate. The two NEW wins (first-class FTS · typed vector k-NN) close former Supabase strengths; the rest are standing structural advantages.

First-class ranked FTS NEW · gate m101
op=list + search:{query,columns,language} → ranked to_tsvector @@ websearch_to_tsquery over concat'd columns, owner-scoped, a typed first-class op. Supabase's textSearch is a single-column filter operator.
Typed vector k-NN NEW · gate m102
op=list + vector:{column,query,k,metric} → ORDER BY col <=>/<->/<#> $vec LIMIT k, capability-gated. Supabase has pgvector but needs a hand-written SQL RPC to expose k-NN ergonomically.
Multi-engine + bring-your-own-DB 9 vs 1
One uniform API over up to 9 engines (pg, mysql, mariadb, mongo, mssql, cockroach, redis, sqlite, http); a tenant can mount their existing DB (tenant_owned). Supabase is Postgres-only. Reproduce: make conformance.
Dense multi-tenancy gate m46
24,888 live tenants @ 2.918 MiB data-plane RSS, 0 standing pools — pool count independent of tenant count (per-request RLS, not pool-per-tenant). A Supabase project is one Postgres DB per project; our per-tenant holding cost is flat.
In-stack OWASP WAF gate m140
ModSecurity v3 + OWASP CRS as the sole public listener (SQLi/XSS/traversal → 403 with CRS rule-IDs) + per-plane network segmentation. Supabase OSS self-host ships neither in-stack.
Footprint floor 5 MB binary
nano idles at 2.008 MiB (4.9 MB binary, 6 ms cold start). Supabase has no comparable single-binary self-host floor at all.

6 · Choose Grobase if / Choose Supabase if

The honest buyer-fit decision in two cards.

Choose Grobase if… self-host · multi-engine · density
you want one backend for any frontend across up to 9 engines (or bring your own DB); pack thousands of tenants on shared infra at flat cost (24,888 @ ~3 MiB, gate m46); start from a 5 MB / 2 MiB binary and grow to a 10K-tenant platform on one codebase, no rewrite; want an in-stack OWASP WAF + plane segmentation; need ranked multi-column FTS or typed vector k-NN as first-class ops; value data residency and independently re-verifiable compliance controls.
Choose Supabase if… managed · certified · polished
you want a fully managed, turnkey hosted Postgres cloud today (sign up, get billed, no ops); a polished hosted Studio out of the box; third-party SOC 2 / ISO 27001 / HIPAA attestation as a hosted product today; globally edge-distributed functions; the largest community + ecosystem; or your workload is Postgres-only forever and you never need a second engine.
